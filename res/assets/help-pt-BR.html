 Uret Patcher possui sete tipos de patches.

 1. Código do Patch (Xposed não é Necessário) :- Este é o código do Patch da aplicação, quando você atualiza o aplicativo, então antes de atualizar 'Desfaça o Patch' do App Caso contrário, depois da instalação terá que fazer o Patch novamente. (Não há necessidade de usar patch novamente, ou seja, somente em Atualizações).

 2. Dalvik Patch (Xposed não é Necessário) :- Este é o cache de patch do aplicativo, mas você pode atualizar facilmente não há necessidade de desfazer o patch, mas reinstalar ou atualizar irá remover o patch. Sendo assim você terá que fazer o patch novamente. (Não há necessidade de usar patch novamente, ou seja, somente em Atualizações).

 3. Esta é a versão mais recente da licença do patch (Xposed não é Necessário) :- (Execute a aplicação pelo menos uma vez antes de corrigí-la) Este é o patch de licença para Apps do Google Play Store ele fará uma implementação no sistema, O App irá trocar os dados falsos e os colocará como dados 'LEGITÍMOS' e enviar nos esses dados para os servidores do Google. Este método é muito eficaz, mas dá uma dor de cabeça nos desenvolvedores desses aplicativos. (Não há necessidade de usar patch novamente, ou seja, somente em Atualizações [Patchear apenas uma vez]).

 4. Patch Nativo (Xposed não é Necessário) :- Códigos de patchs nativos são basicamente recomendado para jogos. (Não há necessidade de usar patch novamente, ou seja, somente em Atualizações).

 5. Salvar Patch (Xposed não é Necessário) :- (Execute o aplicativo pelo menos uma vez antes do patch) Ele fará alterações no dados, ele também poderá ser usado sozinho ou em conjunto com outros patches baseados em requisitos de aplicativo. (Não há necessidade de usar patch novamente, ou seja, somente em Atualizações [Use o patch apenas uma vez]).

 6. Patch de Gancho (Necessário Xposed) :- Hook Patch ou aplicativo de gancho fará alterações dinâmicas no app, mais uma dor de cabeça para desenvolvedores de aplicativos. (Não há necessidade de usar patch novamente, ou seja, somente em Atualizações [Use o patch apenas uma vez]).

 7. Patch do Módulo (Xposed não é Necessário) :- Corrigir verificações do sistema, alguns aplicativos só precisam disto, mas algumas vezes apps Altamente Protegidos precisam interagir com outros patches, ele também pode ser usado sozinho ou em conjunto com outros patches baseados em requisitos de aplicativo. (Não há necessidade de usar patch novamente, ou seja, somente em Atualizações).

 8. Gerar um Patch (Xposed não obrigatório) :- (Execute o aplicativo pelo menos uma vez antes do patch) É alternativo do Patch da Licença, mas funciona sem Xposed, Usa os resultados padrão do sistema em vez de manipulá-lo, como no Patch da Licença. (Não há necessidade de Patchear novamente ie. Em Atualizações [Somente Patchear uma vez]).

 NOTA :- Se alguns aplicativos exigirem mais do que um patch, então somente um tipo de patch com a prioridade mais alta mostrará os detalhes do mesmo (Prioridade :- Patch de licença > Patch de Gancho > Código do Patch > Dalvik Patch > Patch Nativo > Patch do Módulo > Salvar Patch).
O Patch poderá ou não funcionar em atualizações, mas 'PATCH' sempre funcionará na versão que é feita para.

 Se o Uret Patcher não Funciona ou Quebrar (Crash) na execução. verifique na solução de problemas em (Patcher > Menu > Solução de Problemas) Se a solução de problemas não funcionar, Obtenha os Logs em (Patcher > Menu > Registro de Logs) E me envie todos os arquivos de texto no e-mail (Patcher > Menu > Sobre > Me envie um E-mail).
