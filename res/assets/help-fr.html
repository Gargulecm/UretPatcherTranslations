!!..AIDE..!!
 Le Patcheur Utilise 7 Types de Patches.

 1. Code Patch (Module Xposed Non Requis) :- Il Patch le Code de l’Appli, Quand vous Mettez à Jour l’Appli vous devez restaurer l’Appli ('Unpatch') Sinon vous aurez une erreur « Application non installée », une fois la mise à jour installée vous devez la Patcher à nouveau. (Besoin de Restaurer avant et Re-Patcher après chaque Mise à Jour).

 2. Dalvik Patch (Xposed Non Requis) :- Il Patch la Cache de l’Appli, Vous pouvez Mettre à Jour Facilement l’Appli, pas besoin de restaurer l’Appli ('Unpatch') avant la mise à jour. Mais la mise à jour ou la réinstallation va retirer le Patch Donc vous devez Patcher à nouveau l’appli. (Besoin de Re-Patcher après chaque Mise à Jour).

 3. License Patch (Xposed Requis) :- (Lancez l’Appli au moins une fois avant le Patch) Il Patch la License Google et l’Implémentation Intégrée (InApp) ainsi le système est trompé, L’Appli considère les données contrefaites comme des données Authentiques comme s’il s’agissait d’une réponse des serveurs de Google. Il est très efficace mais redoutable pour les développeurs de l’appli. (Pas besoin de Re-Patcher après chaque Mise à Jour [Patcher Une Seule Fois]).

 4. Native Patch (Xposed Non Requis) :- Il Patch le Code natif de l’Appli ,Essentiellement pour les Jeux. (Besoin de Re-Patcher après chaque Mise à Jour).

 5. Save Patch (Xposed Non Requis) :- (Lancez l’Appli au moins une fois avant le Patch) Il enregistre les modifications dans les données sauvegardées (Data) de l’appli. Il peut, aussi bien être appliqué seul, que combiné avec d’autres Patchs, selon les besoins de l’Appli. (Pas besoin de Re-Patcher après chaque Mise à Jour [Patcher Une Seule Fois]).

 6. Hook Patch (Xposed Requis) :- Il trompe l'Application en effectuant Dynamiquement des modifications ,Encore un cauchemar pour les développeurs de l’appli. (Pas besoin de Re-Patcher après chaque Mise à Jour [Patcher Une Seule Fois]).

 7. Module Patch (Xposed Non Requis) :- Modifie les vérifications du Système, Seulement Certaines Applis ont en besoin, Mais quelques fois les Applications Très protégées en ont besoin en combinaison avec d’autres Patchs. Il peut, aussi bien être appliqué seul, que combiné avec d’autres Patchs, selon les besoins de l’Appli. (Besoin de Re-Patcher après chaque Mise à Jour).

 8. Génerer Patch (Xposed Non Requis) :- (Lancez l'appli au moins une fois avant le patch) C'est une alternatif au patch de licence mais fonctionne sans Xposed, ça utilise les résultats par défaut du système au lieu de les manipuler comme dans le Patch de Licence. (Pas besoin de repatcher après les MAJ [Patcher Une Seule Fois]).

 NOTE :- Si certaines Applis nécessitent plus qu’un Patch, seulement le Patch avec la plus haute priorité sera affiché dans Patch Détails (Priorité :- License Patch > Hook Patch > Code Patch > Dalvik Patch > Native Patch > Module Patch > Save Patch).
Le Patch peut fonctionner Ou Pas après une mises à jour de l’appli. Mais le 'PATCH' fonctionneras Toujours sur la Version pour laquelle il à été conçu.

 Si le Patcheur ne fonctionne pas ou Plante chez vous, Vérifiez la section Dépannage (Patcher > Menu > Dépannage) suivez les instructions, Si ca ne fonctionne toujours pas récupérez les Logs (Patcher > Menu > Dump Logs) et envoyez-moi les 3 fichiers texte par Email (Patcher > Menu > A propos > Email Me).
