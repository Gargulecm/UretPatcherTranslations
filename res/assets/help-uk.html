 Патчер має сім типів патчів.

 1. Code Patch (модуль Xposed не потрібен): Патч вносить зміни у початковий код додатку. Перед оновленням додатку до нової версії, скасуйте зроблені патчером зміни, після оновлення застосуйте патч знову. (потрібне повторне застосування патчу після оновлення додатку).

 2. Dalvik Patch (модуль Xposed не потрібен): Патч вносить зміни в dalvik-cache додатку. При оновлені додатку, зміни внесені раніше зберігаються. Але, якщо Ви встановлюєте оновлення шляхом видалення старої версії та встановлення нової, необхідно буде застосувати патч знову, тому що дані патчу не збережуться. (потрібне повторне застосування патчу після оновлення додатку).

 3. License Patch (потрібен модуль Xposed): (Перед застосуванням патчу, потрібно запустити додаток хоча б один раз). Патч вносить зміни в ліцензію Google, InApp сповіщення і систему повідомлень з сервером. Додаток буде приймати "підроблені" дані як "законні", отримані від серверів Google. Це дуже ефективний засіб, але завдає збитків розроникам. (не потрібне повторне застосування патчу після оновлення додатку [одноразове застосування]).

 4. Native Patch (модуль Xposed не потрібен): Патч вносить зміни у початкові ресурси додатку, застосовується в основному для ігор. (потрібне повторне застосування патчу після оновлення додатку).

 5. Save Patch (модуль Xposed не потрібен): (Перед застосуванням патчу, необхідно запустити додаток хоча б один раз). Патч вносить зміни у користувачькі дані додатку. Він може застосовуватись самостійно або разом з іншими патчами, в залежності від потреб. (не потрібне повторне застосування патчу після оновлення додатку [одноразове застосування]).

 6. Hook Patch (потрібен модуль Xposed): Патч перехоплює повідомлення які відправляються на сервер додатку та динамічно вносить зміни. "Великі проблеми для розробників". (не потрібне повторне застосування патчу після оновлення додатку [одноразове застосування]).

 7. Module Patch (модуль Xposed не потрібен): Патч вносить зміни в систему перевірки. Деяким додаткам цього буває досить, проте для додатків з складним захистом, необхідно застосовувати цей патч разом з іншими, в залежності від потреб. (потрібне повторне застосування патчу після оновлення додатку).

 8. Створити патч (Xposed не потрібен) :- (Запустити програму хоча б один раз перед пропатчуванням) це альтернатива пропатчування перевірки ліцензії, але працює без Xposed, патч використовує результати системи по замовчуванню, а не маніпулює ними як при пропатчуванні перевірки ліцензії. (Не потрібно повторно пропатчувати після оновлення [тільки одноразове пропатчування]).

 Примітка: Якщо додаток потребує більше одного патчу, то тільки патч з найвищим пріоритетом буде відображений у деталях патчу (Пріоритет патчів: License Patch > Hook Patch > Code Patch > Dalvik Patch > Native Patch > Module Patch > Save Patch).
Патч завжди буде працювати на тій версії додатку, для котрої він зроблений. На оновленнях чи нових версіях, робота патчу можлива, але не гарантується.

 Якщо патчер не працює або "вилітає", ознайомтесь з інформацією з розділу "Усунення недоліків" (Меню > Усунення недоліків). Якщо проблема не вирішилась, зробіть логи (Меню > Дамп подій) та відправте мені 3 текстових файли на електронну пошту (Меню > Про патчер > Email Me).
