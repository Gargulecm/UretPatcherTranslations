 Patcher Memiliki 7 Macam Patch.

 1. Patch Kode (Membutuhkan Xposed) :- Opsi ini mem-patch Kode Dari Apl,Jika Anda Memperbarui App Maka Sebelum Memperbarui 'Unpatch' App Terlebih Dahulu, Jika Tidak Maka Akan Ada Kesalahan Pemasangan,Setelah Memasang Apl Anda Harus Mem-patch Ulang. (Harus Patch Ulang Ketika Memperbarui).

 2. Patch Dalvik (Tidak Membutuhkan Xposed) :- Opsi Ini Mem-patch Cache dari Apl,Tetapi Anda Bisa Memperbaruinya Dengan Mudah Tanpa Harus Meng-Unpatch Terlebih Dahulu Tetapi Memasang Ulang Atau Memperbarui Apl Akan Menghilangkan Patch,Maka Anda Harus Mem-patch Ulang. (Harus Patch Ulang Ketika Update).

 3. Patch Lisensi License (Membutuhkan Xposed) :- (Jalankan Aplikasi Minimal Satu Kali Sebelum Proses Patch) Opsi Ini Mem-patch Penerapan Lisensi InApp Google Dan System Dialihkan Supaya App Akan Menganggap Data Palsu Sebagai Data 'RESMI' Seolah-olah Respon Dari Server Google.Ini Sangat Efektif,Tetapi Menyakitkan Pengembang App. (Tidak Harus Untuk Mem-patch Ulang Saat Memperbarui [Hanya Sekali Patch]).

 4. Patch Asli (Tidak Membutuhkan Xposed) :- Opsi ini Mem-patch Kode Asli,Pada Dasarnya Untuk Game. (Harus Patch Ulang Ketika Memperbarui).

 5. Patch Aman (Tidak Membutuhkan Xposed) :- (Jalankan Apl Minimal Satu Kali Sebelum Proses Patch) Opsi Ini Membuat Perubahan Pada Data, Ini Juga Dapat Digunakan Tersendiri Ataupun Digabung Dengan Patch Lain Berdasarkan Kebutuhan Apl. (Tidak Harus Untuk Patch Ulang Ketika Memperbarui [Hanya Sekali Patch].

 6. Patch Pengalihan (Membutuhkan Xposed) :- Opsi ini Mengalihkan Apl Secara Dinamis Membuat Perubahan Pada Apl,Sekali Lagi Ini Menyebalkan Bagi Pengembang Apl. (Tidak Harus Untuk Patch Ulang Ketika Memperbarui [Hanya Sekali Patch].

 7. Patch Modul (Tidak Membutuhkan Xposed) :- Memperbaiki Pengecekan System,Beberapa Apl Hanya Membutuhkan ini Tetapi Terkadang Apl Yang Diproteksi Ketat Membutuhkan Ini Dengan Gabungan Patch Yang Lainnya,Ini Juga Bisa digunakan Secara Tersendiri Atau Digabungkan Dengan Patch Lain Berdasarkan Kebutuhan Apl. (Harus Patch Ulang Ketika Memperbarui).

 8. Hasilkan Patch (Tidak dibutuhkan Xposed) :- (Jalankan aplikasi setidaknya satu kali) Ini mengubah patch lisensi tetapi dapat bekerja tanpa Xposed, Ini menggunakan hasil lisensi sistem default alih-alih memanipulasinya seperti pada Patch Lisensi. (Tidak perlu mem-patch lagi ketika update aplikasi [Hanya Patch Sekali].

 CATATAN :- Jika Beberapa Apl Membutuhkan Lebih Dari Satu Patch Maka Patch Dengan Prioritas Tertinggi Saja Yang Ditampilkan Di Detil Patch (Prioritas :- Patch Lisensi > Patch Pengalihan > Patch Kode > Patch Dalvik > Patch Asli > Patch Modul > Patch Aman).
Patch Bisa Saja Berfungsi Ataupun Tidak Saat Pembaruan Tetapi 'PATCH' Akan Selalu Berfungsi Pada Versi Apl Yang Patch Dibuat Untuk Versi Tersebut.

 Jika Patcher Tidak Berfungsi Atau Crash Harap Cek Troubleshooting (Patcher > Menu > Troubleshooting) Ikuti Petunjuk,Jika Masih Tidak Berfungsi Maka Buatlah Log (Patcher > Menu > Dump Log) Dan Kirim Semua File Teks Melalui Email (Patcher > Menu > Tentang > Email Saya).
