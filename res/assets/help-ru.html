 Патчер имеет восемь типов патчей.

 1. Code Patch (модуль Xposed не требуется): Патч вносит изменения в исходный код приложения. Перед обновлением приложения до новой версии, отмените сделанные патчером изменения, после обновления примените патч снова. (требует повторного применения патча после обновления приложения).

 2. Dalvik Patch (модуль Xposed не требуется): Патч вносит изменения в dalvik-cache приложения. При обновлении приложения, изменения внесенные ранее сохраняются. Однако, если вы устанавливаете обновление путем удаления старой версии и установкой новой, необходимо будет применить патч снова, т.к. данные патча не сохранятся. (требует повторного применения патча после обновления приложения).

 3. License Patch (требуется модуль Xposed): (Перед применением патча, необходимо запустить приложение по крайней мере один раз) Патч вносит изменения в лицензию Google, InApp уведомления и систему сообщений с сервером. Приложение будет принимать "поддельные" данные как "законные", полученные от Google Servers. Это очень эффективное средство, но наносит ущерб разработчикам. (не требует повторного применения патча после обновления приложения [однократное применение]).

 4. Native Patch (модуль Xposed не требуется): Патч вносит изменения в исходные ресурсы приложения, применяется в основном для игр. (требует повторного применения патча после обновления приложения).

 5. Save Patch (модуль Xposed не требуется): (Перед применением патча, необходимо запустить приложение по крайней мере один раз) Патч вносит изменения в пользовательские данные приложения. Он может быть применен отдельно или вместе с другими патчами, в зависимости от требований. (не требует повторного применения патча после обновления приложения [однократное применение]).

 6. Hook Patch (требуется модуль Xposed): Патч перехватывает сообщения отправляемые на сервер приложения и динамически вносит изменения. "Боль в заднице" для разработчиков. (не требует повторного применения патча после обновления приложения [однократное применение]).

 7. Module Patch (модуль Xposed не требуется): Патч вносит изменения в систему проверки. Некоторым приложениям этого бывает достаточно, однако для приложений со сложной защитой, необходимо применять этот патч наряду с другими, в зависимости от требований. (требует повторного применения патча после обновления приложения).

 8. Generate Patch (модуль Xposed не требуется) :- (Перед применением патча, необходимо запустить приложение по крайней мере один раз). Это аналог License Patch,но работает без Xposed,он использует ресурсы системы, а не манипулирует ими как в License Patch. (не требует повторного применения патча после обновления приложения [однократное применение]).

 Примечание: Если приложение требует более одного патча, то только патч с высшим приоритетом будет отображен в деталях патча (Приоритет патчей: License Patch > Hook Patch > Code Patch > Dalvik Patch > Native Patch > Module Patch > Save Patch).
Патч всегда будет работать на той версии приложения, для которой он сделан. На обновлениях или новых версиях, работа патча возможна, но не гарантируется.

 Если патчер не работает или "вылетает", ознакомьтесь с информацией из раздела "Устранение неполадок" (Меню > Устранение неполадок). Если проблема не разрешилась, сделайте логи (Меню > Журнал) и отправьте мне 3 текстовых файла на эл.почту (Меню > О патчере > Email Me).
