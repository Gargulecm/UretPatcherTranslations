 本破解器有七种破解方式。

 1. Code Patch 程序代码破解（无需Xposed）： 此选项破解程序的代码，当你更新被破解的应用时，你应该首先选择“取消破解”选项，否则会导致新版本无法安装。 在更新被破解的应用后你需要重新应用破解。

 2. Dalvik Patch 代码缓存破解（无需Xposed）： 此选项破解程序的代码缓存，使用此选项，你可以方便地更新被破解的应用，不需要在安装新版本之前选择“取消破解”选项。 在应用更新时破解会被移除，因此你需要在更新应用之后重新破解。

 3. License Patch 授权验证破解（需要Xposed）：（在破解之前，你需要首先运行应用至少一次）此选项可以破解授权验证和应用内购机制。在系统启用Xposed的情况下，应用会收到一条假的，来自Google服务器的“授权有效”的消息。这个破解非常有效，但是会让应用开发者很难受。 此选项在应用更新后仍然有效，因此只需要破解一次。

 4. Native Patch 库文件破解（无需Xposed）： 此选项破解应用的库文件，一般是针对游戏的破解。 需要在应用更新之后重新应用破解。

 5. Save Patch 数据存储破解（无需Xposed）： （在破解之前，你需要首先运行应用至少一次）此选项更改应用的数据存储，根据应用的需求，你有可能只需单独应用此项破解，也可能需要与其他选项配合使用。 无需在应用更新之后再次破解，只需破解一次。

 6. Hook Patch 钩子破解（需要Xposed）: 在启用Xposed时，此选项会对应用进行动态的更改。（又一个让开发者难受的选项...） 无需在应用更新之后再次破解，只需破解一次。

 7. Module Patch 模块破解（无需Xposed）： 此选项修正系统检查机制，有些应用只需要应用这一项破解，但是有的保护比较完善的应用需要同时应用多项破解。根据应用的需求，你有可能只需单独应用此项破解，也可能需要与其他选项配合使用。 在更新被破解的应用后你需要重新应用破解。

 8. Generate Patch 生成补丁（无需Xposed）： （请在应用此破解选项前运行目标程序至少一次。）此方法是授权验证破解的不使用Xposed的替代方案。它通过使用系统返回的默认结果来破解，而不是像授权验证破解一样对授权验证结果进行修改。 （无需在更新目标应用之后重新应用破解）

 注释：如果某些应用需要多于一项破解，你只需应用优先级最高的一项（在应用详请里会显示）。（通用优先级：授权验证破解 > Xposed钩子破解 > 程序代码破解 > 代码缓存破解 > 库文件破解 > 模块破解 > 数据存储破解）
破解有可能对应用的新版本无效，但是“破解”选项对于此破解的目标版本总是有效的。

 如果破解不工作或者破解之后应用无法运行，你应该查看故障排查选项（破解器 > 菜单 > 故障排查）并按照故障排查里的内容去做。如果仍然无效，请收集日志（破解器 > 菜单 > 收集日志），然后把收集到的所有的文本文件发送给我。（破解器 > 菜单 > 关于 > 发送邮件）
